## 📈 各版本详细记录

### 1. BioHebbian v1 / v3 (scale/pv 版本)

**机制**：
```python
self.scale = nn.Parameter(torch.tensor(0.05))  # 缩放因子
self.pv = nn.Parameter(torch.tensor(0.05))      # 群体向量门控
```

**参数统计**：
| 指标 | 数值 |
|------|------|
| 总参数量 | ~0.48M |
| Hebbian参数 | 72 (2参数/层 × 36层) |
| Hebbian占比 | 0.015% |

**训练结果**：
| Epoch | Val Acc |
|-------|---------|
| 50 | 89.8% |
| 100 | 92.5% (v1) / 92.35% (v3) |

**评估**：
- ✅ 参数量少
- ❌ Hebbian参数太少(72个)，更像训练技巧
- ⚠️ 论文价值存疑

---

### 2. BioHebbian v2 (轻量版)

**机制**：与v1相同，但用了更少的层/通道

**参数统计**：
| 指标 | 数值 |
|------|------|
| 总参数量 | **0.13M** |
| 参数减少 | 73% |
| Hebbian参数 | 72 |

**训练结果**：~73% (Epoch 80，中断)

**评估**：
- ✅ 极轻量
- ⚠️ 精度牺牲较大

---

### 3. True Hebbian (11.8% Hebbian)

**机制**：
```python
self.hebbian_w: [in_ch, growth]           # 突触权重
self.hebbian_intra: [growth, growth]       # 通道内调制
self.modulation: 1                         # 调制因子
```

**参数统计**：
| 指标 | 数值 |
|------|------|
| 总参数量 | 539,566 (0.54M) |
| Hebbian参数 | 63,684 |
| Hebbian占比 | 11.80% |

**训练结果**：
| Epoch | Train | Val |
|-------|-------|-----|
| 1 | 40.6% | 47.8% |
| 10 | 68.4% | 81.5% |
| 20 | 74.5% | 86.0% |
| 30 | 77.3% | 88.8% |
| 50 | 82.7% | 90.5% |
| **66** | - | **91.13%** |

**评估**：
- ✅ 参数量适中
- ✅ 准确率最高(91.13%)
- ✅ Hebbian机制真正起效
- ⚠️ 参数量比标准DenseNet多

---

### 4. HebbianHeavy (67% Hebbian)

**机制**：
```python
self.hebbian_w: [in_ch, growth*16]        # 大幅扩大
self.hebbian_inter: [growth*16, growth*16] # 通道间
self.hebbian_intra: [growth*8, growth*8]   # 通道内
self.mod1/2/3: 调制因子
```

**参数统计**：
| 指标 | 数值 |
|------|------|
| 总参数量 | 3,858,502 (3.86M) |
| Hebbian参数 | 2,594,412 |
| Hebbian占比 | **67.24%** |

**训练结果**：
| Epoch | Train | Val |
|-------|-------|-----|
| 1 | 40.6% | 37.8% |
| 3 | 57.5% | **66.3%** (爆发) |
| 10 | 70.7% | 76.1% |
| 20 | 73.5% | 82.9% |
| 30 | 79.5% | 86.3% |
| 50 | 83.2% | 89.6% |
| **75** | - | **90.53%** |

**关键现象**：
- Epoch 3: Val=66.3% > Train=57.5% (**Hebbian正则化效果**)
- 早停于Epoch 75

**评估**：
- ✅ 证明67% Hebbian参数仍能学习
- ✅ Hebbian机制作为主力有效
- ❌ 参数量效率低(8倍参数)
- ❌ 没超过标准DenseNet

---

### 5. VGG风格 True Hebbian (过拟合版本)

**架构**：非DenseNet，类似VGG/ResNet

**参数统计**：
| 指标 | 数值 |
|------|------|
| 总参数量 | 6.36M |
| Hebbian参数 | 1.35M |
| Hebbian占比 | 21.21% |

**训练结果**：
| Epoch | Train | Val |
|-------|-------|-----|
| 1 | 32.6% | 22.7% |
| 5 | 59.3% | 44.3% |
| 10 | 67.9% | 59.0% |
| 20 | 73.3% | 63.9% |
| 30 | 77.0% | 57.7% (过拟合) |
| 40 | 78.8% | 55.3% (崩溃) |

**评估**：
- ❌ 架构问题，VGG风格不适合DenseNet
- ❌ 严重过拟合
- ❌ 6.36M参数太大

```
准确率
  ↑

    │                ╲╱ ╲
```

- Hebbian占比从11.8%→67%: 准确率略降，但机制更纯粹
- 最优点可能在10-20%区间

## 💡 核心结论

### 1. Hebbian机制确实有效
- 所有含Hebbian的版本都能正常训练
- Val > Train的现象说明Hebbian有正则化效果

### 2. Hebbian参数效率
- **11.8% Hebbian**：最佳效果，参数量适中
- **67% Hebbian**：参数效率低，但机制纯粹

### 3. 与标准BP对比
- 准确率相当，无明显优势
- 参数量持平或更多

### 4. 论文价值评估
| 评估项 | 状态 |

| Hebbian机制有效性 | ✅ 已验证 |
